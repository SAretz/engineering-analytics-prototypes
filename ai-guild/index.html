Section 2

Spec-Driven
Design

How we think about using AI for professional software engineering

Vibe Coding

Natural language in. Output out. Iterate by feel.

  • Great for rapid prototyping and throwaway scripts
  • Works well for solo exploration and one-time tasks
  • No written spec — no reviewable, shareable artifact
  • The codebase becomes the de-facto specification by default
  • Intent diverges from implementation over time

"Vibe coding is for code you throw away."

Spec-Driven Design

Front-load the thinking. The specification is the primary artifact.

  • Write the "what and why" before a single line of code
  • The spec is reviewable, evolvable, and version-controlled
  • Foundation, patterns, architecture — by intent, not by accident
  • Reliable, maintainable, modular output
  • Code is the derived artifact — the spec is the source of truth

"Spec-driven design is for code you stand behind."

The Difference

Vibe Coding

  • Output-first
  • No written artifact
  • Iterate by feel
  • Context lives in your head
  • Great for solo & throwaway
  • Hard to review, test, or hand off
  • Intent drifts from implementation
Code you throw away

Spec-Driven Design

  • Understanding-first
  • Spec = primary artifact
  • Decisions are explicit and written
  • Context is shared and version-controlled
  • Built for teams & scale
  • Reviewable, testable, reproducible
  • Intent is anchored in the spec
Code you stand behind

Why Spec-Driven Design

  • We build software that runs Intersport — it has to last
  • Reliable, maintainable and scalable is non-negotiable
  • Good architecture requires intent, not instinct
  • Modularity and replaceability don't happen by accident
  • We need code we can replace — which means code we understand

We needed a process that produces understanding, not just output. We needed a tool that makes that process practical.

Our Intersport Catalog

Extension

Pre-Specify — Multi-Repo Context

Pulls additional repos into the agent's context before the specify step — only when the task spans multiple repos. Keeps context clean when those repos aren't needed.

Preset Override

Git Branch Naming

Replaces SpecKit's default git extension with Intersport's own branch naming convention. Built on the original — demonstrates how easy SpecKit is to adapt.

Two examples from four weeks of work — every extension is available to both companies through the shared catalog.
